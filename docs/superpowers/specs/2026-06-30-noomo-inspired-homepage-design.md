# Noomo 风格沉浸式个人主页设计方案

## 1. 项目目标

将现有“AI 爱好者”静态个人主页重构为沉浸式内容入口。页面借鉴 Noomo Agency 擅长的全屏叙事、玻璃材质、WebGL 场景和滚动镜头语言，但不复刻其具体作品、构图或视觉资产。

主页的首要目标不是展示前端技术，而是：

1. 让访客快速记住“记录真实 AI 实验过程”的个人定位。
2. 引导访客阅读最新实验复盘。
3. 引导访客关注内容账号，形成长期读者关系。

本阶段只定义设计和实施边界，不实现页面。

## 2. 设计原则

- **内容优先**：3D 场景负责建立情绪和叙事，不遮挡核心文案和行动入口。
- **原创意象**：使用“AI 信号核心”，不使用 Noomo 项目中的凤凰、水母、玻璃球等标志性元素。
- **连续叙事**：各区块不是独立卡片堆叠，而是同一个场景随滚动持续演化。
- **克制配色**：深色中性背景和冷白玻璃为主，荧光绿表示实验信号，暖红表示成本或风险，避免单一蓝紫科技风。
- **渐进增强**：WebGL、动效或外部脚本不可用时，正文、导航和链接仍然完整可用。
- **可维护**：保留现有纯静态架构、双语能力和集中配置方式，不引入不必要的框架或后台。

## 3. 核心视觉：AI 信号核心

首屏中央展示一个半透明、持续变化的数据核心。它由玻璃外壳、内部线框、粒子和脉冲光组成，表达“普通 AI 爱好者正在形成中的实验系统”，而不是已经成熟的成功案例。

滚动时，核心依次发生四个状态变化：

1. **形成**：分散粒子汇聚成核心，出现个人定位和最新实验入口。
2. **分流**：核心拆分为三条轨迹，对应 AI 工具实测、AI 内容创作、AI 变现探索。
3. **聚焦**：镜头靠近其中一条信号，过渡到最新复盘内容。
4. **回流**：信号重新汇聚，形成关注入口和平台链接。

鼠标或触控输入只提供轻微视差，不允许用户操作导致文字位置变化或阅读中断。

## 4. 页面结构

### 4.1 固定导航

- 左侧显示“AI 爱好者”品牌标识。
- 右侧保留实验记录、内容方向、关注入口和中英切换。
- 首屏使用透明导航；进入正文后增加半透明背景以保持可读性。
- 所有导航项支持键盘操作，并具备清晰的焦点状态。

### 4.2 首屏：身份信号

- 全视口 WebGL 场景作为背景，不放入卡片容器。
- 文案保持简短：个人定位、当前实验状态、一个主要行动入口。
- 主按钮指向最新复盘，次要入口指向关注区域。
- 首屏底部露出下一段内容，避免形成封闭的宣传页观感。

### 4.3 三条实验轨迹

- 用同一 3D 场景的三条光轨承载现有三个内容方向。
- 每条轨迹只展示标题和一句解释，减少首屏后的阅读负担。
- 滚动到对应阶段时突出当前轨迹，其余轨迹降低亮度，但不完全消失。

### 4.4 最新复盘

- 镜头从抽象场景过渡到更稳定的阅读区域。
- 保留文章标题、摘要和阅读入口。
- “3000 元工具投入”等真实数据作为辅助信号展示，不使用夸张计数动画。
- 正文区域必须在没有 WebGL 的情况下保持相同的信息层级。

### 4.5 关注入口

- 场景中的三条轨迹重新汇聚，形成明确的结束状态。
- 平台入口保持真实链接驱动；未配置的平台继续显示“即将同步”。
- 页脚保留“不贩卖成功学，只分享可验证的真实过程”。

## 5. 技术方案

### 5.1 架构

- 继续使用 `index.html`、`styles.css` 和 `script.js`。
- 通过固定版本的 CDN ESM 模块加载 Three.js。
- 通过固定版本的 GSAP 和 ScrollTrigger 驱动时间线。
- 不增加 npm、打包器、前端框架、后端、数据库或 CMS。

### 5.2 场景边界

Three.js 场景只接收以下输入：

- 标准化滚动进度 `0..1`
- 当前视口尺寸
- 标准化指针位置 `-1..1`
- 性能等级 `high | reduced | fallback`

场景不读取业务文案或平台配置。HTML 区块通过 `data-scene` 标识动画阶段，滚动控制器将页面进度映射到场景时间线。

### 5.3 内容配置

延续 `script.js` 中的 `siteConfig`，集中管理：

- 最新文章标题、摘要和链接
- 平台名称、状态和链接
- 已投入金额等实验数据
- 默认语言
- 视觉性能等级覆盖项

中英文文案继续使用现有 i18n 数据结构。场景不因切换语言而重建。

## 6. 性能与降级

- 桌面端限制设备像素比，避免高分屏产生不必要的渲染成本。
- 移动端减少粒子数量、材质采样和灯光数量，关闭高成本后期处理。
- 页面进入后台时暂停渲染；恢复可见时继续。
- `prefers-reduced-motion: reduce` 下关闭镜头穿越、持续旋转和强视差，展示稳定的静态核心。
- WebGL 初始化失败、脚本超时或 CDN 不可用时显示本地预渲染主视觉。
- 降级状态不得隐藏导航、正文、语言切换或平台链接。
- 首屏加载期间使用与最终场景构图一致的静态占位，避免大幅布局偏移。

## 7. 无障碍要求

- Canvas 标记为装饰性元素，不承载唯一信息。
- 页面保持语义化标题层级和原生链接行为。
- 所有功能可通过键盘完成。
- 交互元素具备可见焦点状态，正文和按钮满足可读对比度。
- 不使用自动播放声音。
- 动画不能产生频闪，也不能阻止用户正常滚动。
- 中英文切换后更新文档语言，并确保长英文标题不会溢出。

## 8. 验收标准

### 功能

- 导航锚点、中英切换、最新文章入口和平台链接均可使用。
- 四段滚动叙事按形成、分流、聚焦、回流顺序运行。
- 鼠标视差不会改变内容布局或阻碍点击。
- 页面无 JavaScript 时仍可阅读主要内容。

### 视觉

- 1440 x 900 和 390 x 844 视口下无文字遮挡、按钮溢出或异常横向滚动。
- 首屏明确出现品牌、定位、行动入口和下一段内容提示。
- Canvas 首屏非空，滚动前后场景构图存在可观察变化。
- 页面不依赖卡片嵌套、装饰性渐变球或单一蓝紫配色建立层次。

### 性能与兼容

- 移动端不会使用桌面端完整粒子和后期处理配置。
- 减少动态效果模式下不存在持续运动或镜头穿越。
- WebGL 失败时自动显示静态主视觉，所有链接保持可用。
- Chrome、Edge、Safari 当前稳定版本具备可用体验。

### 验证方式

- 使用 Playwright 检查桌面和移动视口截图。
- 通过 Canvas 像素检查确认首屏不是空白画布。
- 在滚动起点、中段和终点分别截图，验证场景状态变化。
- 使用键盘完成导航、语言切换和主要链接访问。
- 使用 Lighthouse 检查严重无障碍问题和移动端首屏可用性，不以追求满分为代价削弱设计目标。

## 9. 明确不在范围内

- 不增加登录、评论、订阅后台或内容管理系统。
- 不制作可复用的通用 3D 编辑器。
- 不购买或引入第三方 3D 模型。
- 不在本阶段重写现有文章内容。
- 不直接复制 Noomo Agency 的源代码、资产或具体视觉作品。
